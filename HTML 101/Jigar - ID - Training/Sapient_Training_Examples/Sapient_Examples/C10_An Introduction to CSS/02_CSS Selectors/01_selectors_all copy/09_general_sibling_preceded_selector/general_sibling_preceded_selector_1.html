Element1~Element2 / General Sibling / Preceded Selector
Meaning / Definition:
  • The element1~element2 selector matches occurrences of element2 that are preceded by element1.
  • Both elements must have the same parent, but element2 does not have to be immediately preceded by element1.
    		•
    Scenario / Use : Select and style every <li> element that are preceded by <p> with the same parent.
    Syntax : elementName1~elementName2 {css properties : values ; }
    Example : elementName1~elementName2 | p~li { text-decoration : underline ; }

    Broswer Support :
    The element~element / adjacent sibling selector is supported in all major browsers.

    Note: For element~element selector to work in IE8 and earlier, a <!DOCTYPE> must be declared.
